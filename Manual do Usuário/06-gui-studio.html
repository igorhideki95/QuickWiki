Capítulo 06

QuickWiki Studio

Use a GUI local do QuickWiki para operar o sistema com menos atrito, mais contexto visual e atalhos diretos para o espelho gerado.

Objetivo

Apresentar a interface visual oficial do projeto para usuários que preferem uma operação mais guiada.

Como abrir

python run_scraper.py --gui

Se quiser usar outra porta:

python run_scraper.py --gui --gui-port 8899

O que existe na tela

  • estado atual do sistema e resumo do último output;
  • formulário de execução com perfil, seed, pasta de saída e limites principais;
  • descrição do perfil selecionado com seed padrão;
  • configurações avançadas para retries, checkpoint, bootstrap e logs;
  • atalhos para espelho, admin, resumo JSON e manual.

Fluxo recomendado

  1. Valide os perfis.
  2. Escolha o perfil correto.
  3. Defina uma amostra pequena com `Max. paginas`.
  4. Inicie o crawl e acompanhe os logs.
  5. Revise o `exit code` e o resumo.
  6. Abra o espelho pelos atalhos da direita.

Observações importantes

  • A GUI não substitui o motor do scraper; ela usa o mesmo `run_scraper.py` por trás.
  • Enquanto houver uma execução ativa, a interface bloqueia outra tentativa.
  • O output ativo da tela acompanha a execução mais recente iniciada pela GUI.
Dica prática

Mesmo com a interface visual pronta, continue começando por crawls pequenos. Isso acelera revisão, reduz retrabalho e deixa a operação mais previsível.
